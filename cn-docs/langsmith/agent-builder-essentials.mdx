## 工具

工具让您的智能体能够与您的应用程序和服务进行交互。您的智能体可以发送电子邮件、创建日历事件、发布消息、搜索网络等等。您可以从 Gmail、Slack、Google Calendar、GitHub 等众多内置工具中进行选择。

有关完整列表，请参阅[支持的工具](/langsmith/agent-builder-tools)。

## 触发器

触发器定义了您的智能体应在何时开始运行。您可以将智能体连接到外部工具或基于时间的计划，使其能够自动响应消息、电子邮件或周期性事件。

以下是触发智能体的一些常用方式：

<CardGroup cols={3}>
  <Card title="Slack" icon="slack">
    当特定 Slack 频道收到消息时，激活您的智能体。
  </Card>
  <Card title="Gmail" icon="envelope">
    当收到电子邮件时，触发您的智能体。
  </Card>
  <Card title="Cron 计划任务" icon="clock">
    让您的智能体基于时间计划运行，以处理周期性任务。
  </Card>
</CardGroup>

## 记忆与更新

您的智能体会随着时间的推移变得更智能。它们会记住先前对话中的重要信息，并能自我更新以更好地工作。

- **记忆**：智能体会记住过去交互中的相关细节，以便在未来的对话中做出更好的决策。
- **自我更新**：智能体可以添加新工具、移除不需要的工具或调整其指令，以改进其工作方式。
- **保持不变的部分**：智能体无法更改其名称、描述或启动它们的触发器。

## 自定义模型

Agent Builder 支持自定义模型。您可以通过编辑智能体的设置，为特定智能体覆盖默认的 Anthropic 或 OpenAI 模型：

1.  在 [LangSmith UI](https://smith.langchain.com) 中，导航到您要编辑的智能体。
2.  点击右上角的 <Icon icon="gear" /> 设置图标。
3.  在 **模型** 部分，选择 **+ 添加自定义模型**。
4.  输入模型 ID、显示名称、基础 URL 以及 API 密钥名称和值。
5.  点击 **保存**。

<Note>
  自定义模型的性能可能不如内置模型。在生产环境中使用自定义模型之前，请对其进行测试。
</Note>

## 子智能体

通过将大型任务分解为更小、更专业的助手来构建复杂的智能体。您可以将子智能体视为一个专家团队——每个子智能体处理工作的特定部分，同时与您的主智能体协同工作。

这种方法使得构建复杂的系统变得更加容易。您无需让一个智能体尝试完成所有事情，而是可以拥有各自擅长任务特定部分的专业助手。

以下是您可能使用子智能体的一些方式：

- **拆分为子任务**：让一个智能体获取数据，另一个智能体进行总结，第三个智能体格式化结果。
- **专用工具**：根据不同的需求，为不同的智能体提供不同的工具访问权限。
- **独立工作**：让子智能体独立工作，然后将它们的结果带回主智能体。

## 人在回路

保持对重要决策的控制。您可以设置您的智能体，使其在执行某些操作之前暂停并请求您的批准。这确保了您的智能体可以自动处理大多数任务，同时您保留监督权。

### 设置审批步骤

<Steps>
  <Step title="选择一个工具">
    在设置智能体时，选择您希望在运行前进行审查的工具或操作。
  </Step>
  <Step title="开启审批">
    找到该工具的审批选项并将其打开。
  </Step>
  <Step title="智能体等待您">
    当您的智能体执行到该步骤时，它将暂停并等待您的批准，然后才会继续。
  </Step>
</Steps>

### 智能体暂停时您可以做什么

当您的智能体停下来请求批准时，您有三个选项：

<CardGroup cols={3}>
  <Card title="接受" icon="check">
    给予绿灯，让您的智能体按计划继续执行。
  </Card>
  <Card title="编辑" icon="pen-to-square">
    在允许智能体继续之前，修改其消息或参数。
  </Card>
  <Card title="发送反馈" icon="comment">
    分享反馈以帮助您的智能体学习和改进。
  </Card>
</CardGroup>
