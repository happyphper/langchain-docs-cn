**LangGraph v1 是一个专注于稳定性的智能体运行时版本。** 它保持了核心的图 API 和执行模型不变，同时改进了类型安全性、文档和开发者体验。

它旨在与 [LangChain v1](/oss/releases/langchain-v1)（其 `create_agent` 构建于 LangGraph 之上）无缝协作，因此你可以从高级用法开始，并在需要时深入到底层的精细控制。

<CardGroup cols={1}>
    <Card title="稳定的核心 API" icon="diagram-project">
        图的基本元素（状态、节点、边）以及执行/运行时模型保持不变，使得升级过程简单直接。
    </Card>
    <Card title="默认的可靠性" icon="database">
        具备检查点、持久化、流式处理和人工介入的持久化执行功能，仍然是首要特性。
    </Card>
    <Card title="与 LangChain v1 无缝集成" icon="link">
        LangChain 的 `create_agent` 运行在 LangGraph 上。使用 LangChain 快速上手；切换到 LangGraph 进行自定义编排。
    </Card>
</CardGroup>

升级方法：

<CodeGroup>
```bash pip
pip install -U langgraph
```
```bash uv
uv add langgraph
```
</CodeGroup>

## `create_react_agent` 的弃用

LangGraph 的 @[`create_react_agent`] 预构建功能已被弃用，转而推荐使用 LangChain 的 @[`create_agent`]。后者提供了更简洁的接口，并通过引入中间件提供了更大的自定义潜力。

* 关于新的 @[`create_agent`] API 的信息，请参阅 [LangChain v1 发布说明](/oss/releases/langchain-v1#create-agent)。
* 关于从 @[`create_react_agent`] 迁移到 @[`create_agent`] 的信息，请参阅 [LangChain v1 迁移指南](/oss/migrate/langchain-v1#create-agent)。
