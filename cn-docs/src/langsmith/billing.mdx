## 为您的账户设置账单

<Note>
在使用本指南之前，请注意以下几点：

- 如果您对[企业版](https://www.langchain.com/pricing)计划感兴趣，请[联系销售](https://www.langchain.com/contact-sales)。本指南仅适用于我们的自助服务账单计划。
</Note>

要为您的 LangSmith 组织设置账单，请导航至 **设置** 下的[账单和使用情况](https://smith.langchain.com/settings/payments)页面。根据您组织的设置，有不同的设置指南：

- [开发者计划](#developer-plan%3A-set-up-billing-on-your-personal-organization)
- [Plus 计划](#plus-plan%3A-set-up-billing-on-a-shared-organization)

### 开发者计划：在您的个人组织上设置账单

个人组织在添加信用卡之前，每月限制为 5,000 条追踪记录。要添加信用卡：

1. 点击 **添加信用卡以解除追踪限制**。
2. 添加您的信用卡信息。
3. 完成后，您将不再受限于 5,000 条追踪记录的速率限制，并且超出部分将按照[定价](https://www.langchain.com/pricing-langsmith)页面上指定的费率收费。

### Plus 计划：在共享组织上设置账单
团队组织初始每月可获得 10,000 条追踪记录。超出部分将按照[定价](https://www.langchain.com/pricing-langsmith)页面上指定的费率收费。

<Note>
您手动创建的新组织必须使用 Plus 计划。如果您看到需要升级到 Plus 才能使用此组织的消息，请按照以下步骤操作。
</Note>

1. 点击 **升级到 Plus**。
2. 根据需要邀请成员加入您的组织。
3. 输入您的信用卡信息。然后，输入业务信息、发票电子邮件和税务 ID。如果此组织属于某个企业，请勾选 **这是企业** 复选框并相应填写信息。更多信息，请参阅[更新您的信息部分](#update-your-information)。

## 更新您的信息（仅限付费计划）

要更新您的 LangSmith 组织的业务信息，请前往 **设置** 下的[账单和使用情况](https://smith.langchain.com/settings/payments)页面。

### 发票电子邮件

要更新发票的电子邮件地址，请按照以下步骤操作：

1. 导航到 **计划和账单** 选项卡。
2. 在付款方式下方找到显示当前发票电子邮件的部分。
3. 在提供的字段中输入新的发票电子邮件地址。
4. 新的电子邮件地址将自动保存。

您未来的所有发票都将发送到更新后的电子邮件地址。

### 业务信息和税务 ID

<Note>
在某些司法管辖区，LangSmith 需要征收销售税。如果您是企业，提供您的税务 ID 可能使您有资格获得销售税豁免。
</Note>

要更新您组织的业务信息，请按照以下步骤操作：

1. 导航到 **计划和账单** 选项卡。
2. 在发票电子邮件部分下方，您会找到一个标记为 **企业** 的复选框。
3. 如果您的组织属于某个企业，请勾选 **企业** 复选框。
4. 将出现一个业务信息部分，允许您输入或更新以下详细信息：
   - 企业名称
   - 地址
   - 适用司法管辖区的税务 ID
5. 选择国家/地区后，将出现适用司法管辖区的税务 ID 字段。
6. 输入必要信息后，点击 **保存** 按钮以保存更改。

这确保您的业务信息在计费和税务方面保持最新和准确。

### 了解您当前的使用情况

任何优化过程的第一步都是了解当前的使用情况。LangSmith 提供了两种方法来实现这一点：[使用量图表](#usage-graph) 和 [发票](#invoices)。

LangSmith 使用量是按工作区衡量的，因为工作区通常代表开发环境（如示例所示）或组织内的团队。

#### 使用量图表

使用量图表让您可以检查已消耗了多少基于使用量的定价指标。它不直接显示支出（您稍后将在发票草稿中查看）。

导航至 **设置** -> **计费与使用量** -> **使用量图表**。

LangSmith 对以下几种使用量指标收费：

- LangSmith 追踪（基础费用）：追踪您发送到 LangSmith 的所有追踪记录。
- LangSmith 追踪（扩展数据保留升级）：追踪所有同时启用了 400 天扩展数据保留的追踪记录。
- LangSmith 部署运行：追踪已部署的 LangGraph 智能体的端到端调用。

有关追踪的更多详细信息，请参阅 [数据保留概念文档](/langsmith/administration-overview#data-retention)。有关部署运行的更多详细信息，请参阅 [LangSmith 部署计费](#langsmith-deployment-billing)。

#### 发票

要了解您的使用量如何转化为支出，请导航到 **发票** 选项卡。屏幕上显示的第一张发票是您当前月份的发票草稿，其中显示了您本月迄今为止的累计支出。

<Note>
LangSmith 的使用量图表和发票使用术语 `tenant_id` 来指代工作区 ID。它们是可互换的。
</Note>

### 设置使用量限制

![P2usagelimitsempty v2](/langsmith/images/p2usagelimitsempty-v2.png)

#### 为工作区设置支出限制

1.  要设置限制，请导航至 **设置** -> **计费与使用量** -> **使用量限制**。
2.  为您选定的工作区输入支出限制。LangSmith 将确定适当数量的基础追踪和扩展追踪限制以匹配该支出。追踪限制包括您的计划附带的免费追踪额度（详情请参阅 [定价页面](https://smith.langchain.com/settings/payments)）。

<Note>
**仅适用于拥有多个工作区的组织**：为简化起见，LangSmith 仅将免费追踪额度纳入**第一个工作区**的成本计算中。实际上，免费追踪额度可以被任何工作区“消耗”。因此，尽管对于多工作区组织来说，工作区级别的支出限制是近似值，但组织级别的支出限制是绝对的。
</Note>

#### 配置追踪层级分布
LangSmith 有两个追踪层级：基础追踪和扩展追踪。基础追踪具有基础保留期且是短期的（14 天），而扩展追踪具有扩展保留期且是长期的（400 天）。更多信息，请参阅 [数据保留概念文档](/langsmith/administration-overview#data-retention)。

在 **默认数据保留** 标签下方选择一个选项，以设置所需的默认追踪层级。所有追踪在注册时默认都将采用此层级。请注意，由于扩展追踪的成本高于基础追踪，选择 **扩展** 作为默认数据保留选项将导致计费周期内允许的追踪总数减少。默认情况下，更新此设置仅适用于未来传入的追踪。要将其应用于工作空间中的所有现有追踪，请勾选复选框。

如果默认数据保留设置为 **基础**，您可以选择使用滑块在基础追踪和扩展追踪之间分配追踪限额。LangSmith 会自动为此分配提供建议，但您可以根据需要进行调整。例如，如果您正在运行大量可能将追踪升级为扩展的自动化或其他功能，您可能希望增加扩展追踪限额。要查看可能升级追踪的功能的完整列表，[请参阅此处](https://docs.langchain.com/langsmith/administration-overview#how-it-works:~:text=Data%20retention%20auto%2Dupgrades)。

<Note>
达到扩展数据保留限额后，可能会导致除追踪之外的其他功能停止工作。如果您计划使用此功能，请阅读更多关于其[功能和副作用](/langsmith/administration-overview#side-effects-of-extended-data-retention-traces-limit)的信息。
</Note>

### 管理追踪的其他方法

#### 更改项目级默认保留设置

数据保留设置可按追踪项目进行调整。

导航至 **项目** > ***您的项目名称*** > 选择 **保留** 并选择所需的默认保留设置。这只会影响 **未来追踪** 的保留（和定价）。

![P1projectretention](/langsmith/images/p1projectretention.png)

#### 对一定比例的追踪应用扩展数据保留

您可能不希望所有追踪都在 14 天后过期。您可以通过创建[自动化规则](/langsmith/rules)，自动延长符合某些条件的追踪的保留期。您可能希望对特定类型的追踪应用扩展数据保留，例如：

*   所有追踪的 10%：用于一般分析或长期趋势分析。
*   出错的追踪：用于彻底调查和调试问题。
*   具有特定元数据的追踪：用于对特定功能或用户流程进行长期检查。

要进行配置：

1.  导航至 **项目** > ***您的项目名称*** > 选择 **+ 新建** > 选择 **新建自动化**。
2.  为您的规则命名，并可选择应用过滤器或采样率。有关配置过滤器的更多信息，请参阅[过滤技术](/langsmith/filter-traces-in-application#filter-operators)。

<Note>
当自动化规则匹配[追踪](/langsmith/observability-concepts#traces)内的任何[运行](/langsmith/observability-concepts#runs)时，该追踪内的所有运行都将升级为保留 400 天。
</Note>

例如，这是将所有追踪的 10% 保留用于扩展数据保留的预期配置：

![P2sampletraces](/langsmith/images/P2SampleTraces.png)

如果您希望出于数据收集目的，将一部分追踪保留 **超过 400 天**，您可以创建另一个运行规则，将某些运行发送到您选择的数据集。数据集允许您存储追踪的输入和输出（例如，作为键值数据集），并且将无限期持久保存，即使在追踪被删除之后。

### LangSmith 部署计费

除了追踪之外，LangSmith 还会对通过 LangSmith 部署（原 LangGraph 平台）部署的智能体进行计费。

- **部署运行（Deployment Runs）**：指已部署的 LangGraph 智能体（agent）的一次端到端调用，每次计费 0.005 美元。单个智能体执行过程中的节点和子图不单独计费。调用其他 LangGraph 智能体时，费用由托管被调用智能体的部署单独计收。在使用带中断功能的人机协同（human-in-the-loop）时，中断后恢复将创建一次独立的部署运行。
- **部署运行时间（Deployment Uptime）**：您还需为部署数据库处于活动状态并持续保存状态的时间付费。有关不同部署类型（开发版与生产版）的运行时间成本，请参阅[定价页面](https://www.langchain.com/pricing)。

对于高使用量的部署需求，请[联系我们的销售团队](https://www.langchain.com/contact-sales)以讨论定制定价方案。
