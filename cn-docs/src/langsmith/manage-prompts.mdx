LangSmith 提供了多种工具来帮助您有效管理 [_提示词_](/langsmith/prompt-engineering-concepts)。本页描述了以下功能：

- [提交标签](#commit-tags)，用于版本控制和环境管理。
- [Webhook 触发器](#trigger-a-webhook-on-prompt-commit)，用于在提示词更新时自动化工作流。
- [公共提示词中心](#public-prompt-hub)，用于发现和使用社区创建的提示词。

## 提交标签

[_提交标签_](/langsmith/prompt-engineering-concepts#tags) 是引用您提示词版本历史中特定 [_提交_](/langsmith/prompt-engineering-concepts#commits) 的标签。它们帮助您标记重要版本，并控制在不同环境中运行哪个版本。通过在代码中引用标签而非提交 ID，您可以在不修改代码本身的情况下更新所使用的版本。

每个标签精确引用一个提交，但您可以将标签重新分配以指向不同的提交。

<Note>
**请勿与资源标签混淆**：提交标签专用于提示词版本控制，引用提示词历史中的单个提交。[资源标签](/langsmith/set-up-resource-tags) 是用于组织工作空间资源（如项目、数据集和提示词）的键值对。虽然两者可能使用类似的命名约定（如 `prod` 或 `staging`），但提交标签控制提示词的 **哪个版本** 运行，而资源标签帮助您 **组织和筛选** 工作空间中的资源。
</Note>

### 创建标签

要创建标签，请导航到提示词的 **提交** 标签页。点击您想要标记的提交旁边的标签图标。点击 **新建标签** 并输入标签名称。

![提交标签页](/langsmith/images/commits-tab.png) ![创建新提示词标签](/langsmith/images/create-new-prompt-tag.png)

### 移动标签

要将标签指向不同的提交，请点击目标提交旁边的标签图标，然后选择您想要移动的标签。这将自动更新标签以指向新的提交。

![移动提示词标签](/langsmith/images/move-prompt-tag.png)

### 删除标签

要删除标签，请点击您想要删除的标签旁边的删除图标。这将完全删除该标签，它将不再与任何提交关联。

### 在代码中使用标签

标签提供了一种在代码中引用提示词特定版本的稳定方式。您可以直接引用标签，而不是使用提交哈希，这样可以在不更改代码的情况下更新引用。

以下是在 Python 中通过标签拉取提示词的示例：

```python
prompt = client.pull_prompt("joke-generator:prod")
# 如果 prod 标签指向提交 a1b2c3d4，这等同于：
prompt = client.pull_prompt("joke-generator:a1b2c3d4")
```

有关如何在代码中使用提示词的更多信息，请参阅 [以编程方式管理提示词](/langsmith/manage-prompts-programmatically)。

## 在提示词提交时触发 Webhook

您可以配置一个 Webhook，以便在每次向提示词提交更改时触发。

一些常见的用例包括：

* 在提示词更新时触发 CI/CD 流水线。
* 将提示词与 GitHub 仓库同步。
* 通知团队成员关于提示词的修改。

### 配置 Webhook

从左侧边栏或应用程序主页导航到 **提示词** 部分。在右上角，点击 `+ Webhook` 按钮。

添加 Webhook URL 和任何必需的请求头。

<Note>
每个工作空间只能配置一个 Webhook。如果您希望每个工作空间配置多个，或为每个提示词设置不同的 Webhook，请在 [LangChain 论坛](https://forum.langchain.com/) 中告诉我们。
</Note>

要测试您的 Webhook，请点击 **发送测试通知** 按钮。这将向您提供的 Webhook URL 发送一个包含示例负载的测试通知。

示例负载是一个包含以下字段的 JSON 对象：

- `prompt_id`：已提交提示词的 ID。
- `prompt_name`：已提交提示词的名称。
- `commit_hash`：提示词的提交哈希。
- `created_at`：提交日期。
- `created_by`：提交作者。
- `manifest`：提示词的清单。

### 触发 Webhook

向提示词提交更改以触发您配置的 Webhook。

#### 使用 Playground

如果您在 Playground 中执行此操作，系统将提示您取消选择您希望避免触发的 Webhook。

![在 Playground 中提交提示词](/langsmith/images/commit-prompt-playground.png)

#### 使用 API

如果您通过 API 提交，可以通过将 `skip_webhooks` 参数设置为 `true` 或设置为要忽略的 Webhook ID 数组来指定跳过触发 Webhook。更多信息请参阅 [API 文档](https://api.smith.langchain.com/redoc#tag/commits/operation/create_commit_api_v1_commits__owner___repo__post)。

## 公共提示词中心

LangSmith 的公共提示词中心是一个由 LangChain 社区创建的提示词集合，可供您参考使用。

<Note>
请注意，提示词由用户生成且未经验证。LangChain 不审查或认可公共提示词，使用风险自负。使用提示词中心需遵守我们的 [服务条款](https://www.langchain.com/terms-of-service)。
</Note>

导航到左侧边栏的 **提示词** 部分，然后点击 **浏览 LangChain Hub 中的所有公共提示词**。

在这里，您将找到 LangChain Hub 中所有公开列出的提示词。您可以按名称、句柄、用例、描述或模型搜索提示词。您可以将提示词分叉到您的个人组织，查看提示词的详细信息，并在 Playground 中运行提示词。您可以使用 SDK [将任何公共提示词拉取到您的代码中](/langsmith/manage-prompts-programmatically)。

要查看与您工作空间关联的提示词，请访问边栏中的 **提示词** 标签页。

![提示词标签页](/langsmith/images/prompts-tab.png)
