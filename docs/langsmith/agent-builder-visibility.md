---
title: 智能体可见性
description: 了解 Agent Builder 中私有智能体与工作区智能体的区别。
---
Agent Builder 支持两种智能体可见性模式：

- **私有智能体**：仅创建者可见。适用于个人工作流和实验。
- **工作区智能体**：在工作区内共享。适用于团队工作流或希望与他人共享的智能体。

## 差异对比

| 功能特性 | 私有智能体 | 工作区智能体 |
| --- | --- | --- |
| **所有权与访问权限** | 仅对您自己可见 | 同一 LangSmith 工作区内的所有成员均可见 |
| **OAuth 认证** | OAuth 凭据范围限定于您个人 | OAuth 凭据范围限定于每个用户；新用户克隆工作区智能体时，必须为所选工具重新进行身份验证 |
| **密钥** | 使用工作区范围的 LangSmith 密钥 | 使用工作区范围的 LangSmith 密钥（与私有智能体相同） |

## 工作区范围智能体详情

虽然工作区范围的智能体是共享的，但其中部分详情是公开的，而其他部分则是私有的。

### 线程/聊天历史

线程始终是用户范围的。因此，即使智能体是工作区范围的，在该智能体内创建的聊天历史也始终是私有的，仅对创建它们的特定用户可访问。

### 系统提示、工具、子智能体

系统提示、所选工具和子智能体在工作区范围的智能体上是公开的。用户无法修改原始工作区范围智能体上的这些字段，但克隆智能体后可以进行更改。

### 触发器

工作区范围智能体上的触发器类型是公开的（例如，收到 Slack 消息），但触发器的具体连接（例如，Slack 频道或 Gmail 地址）不会被共享。这样，用户在克隆智能体时知道要使用哪种触发器，但无法未经授权访问原始用户已设置的任何连接。
