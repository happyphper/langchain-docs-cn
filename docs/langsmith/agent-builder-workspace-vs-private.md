Agent Builder 支持两种可见性模式：

- **私有智能体**：仅创建者可见。适用于个人工作流和实验。
- **工作区智能体**：在工作区内共享。适用于团队工作流，或您希望与他人共享的智能体。

## 差异

- **所有权和访问权限**：私有智能体仅对您自己可见；工作区智能体对同一 LangSmith 工作区内的其他任何人都可见。
- **工具认证**：
    - **OAuth**：两种模式都支持 OAuth 和基于密钥的工具。OAuth 凭证始终限定于特定用户，因此工作区智能体无法共享 OAuth 令牌，新用户在克隆工作区智能体时必须使用所选工具重新进行身份验证。
    - **密钥**：由于密钥限定于工作区，工作区智能体和私有智能体都将使用相同的 LangSmith 密钥。

## 公开与私有的区别

### 线程/聊天历史

线程始终限定于用户，因此即使智能体是工作区范围的，在该智能体内创建的聊天历史也始终是私有的，并且只能由创建它们的特定用户访问。

### 系统提示、工具、子智能体

系统提示、所选工具和子智能体在工作区范围的智能体上是公开的。用户无法修改原始工作区范围智能体上的这些字段，但克隆智能体后可以进行更改。

### 触发器

工作区范围智能体上的触发器类型是公开的（例如，收到 Slack 消息），但与触发器的具体连接（例如 Slack 频道或 Gmail 地址）不会被共享。这样，用户在克隆智能体时知道要使用什么触发器，但无法未经授权访问原始用户已设置的任何连接。
