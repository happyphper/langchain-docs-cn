{product} 为不同版本提供不同级别的支持，可能包括新功能、错误修复或安全补丁。

## 支持级别

共有四种支持级别：

- 活跃支持
- 关键支持
- 生命周期终止
- 已弃用

### 活跃支持

其中 N 代表最新的次要版本（例如 0.3、0.4 等）。

当前次要版本 (N) 获得全面支持，包括：
- 新功能和能力
- 错误修复和回归问题修复
- 安全补丁
- 生活质量改进
- 范围明确的高置信度变更

### 关键支持

上一个次要版本 (N-1) 获得有限支持：
- 关键安全修复
- 安装修复
- 不提供新功能或常规错误修复
- 当发布新的次要版本时，从活跃支持过渡而来

### 生命周期终止

早于 N-2 的版本（N-2、N-3 等）不提供支持：
- 没有新的补丁版本发布
- 不修复错误，包括已知错误
- 不提供安全更新
- 用户应升级到受支持的版本

### 已弃用

不再维护的版本：
- 第一个稳定版本之前的所有版本
- 已明确弃用的版本
- 不提供任何支持或维护

## 版本支持策略

{product} 对次要版本遵循 N-2 支持策略：

- **N (当前版本)**：活跃支持
- **N-1**：关键支持
- **N-2 及更早版本**：生命周期终止

### 次要版本支持

次要版本包含新功能和能力，并根据 N-2 策略提供支持。当我们提及次要版本（例如 v0.3）时，始终指其最新的可用补丁版本（v0.3.x）。

### 补丁版本发布

在每个版本的支持窗口期内：

- **活跃支持**：定期发布包含错误修复、回归问题修复和新功能的补丁版本
- **关键支持**：仅针对安全和安装相关的关键修复发布安全补丁
- **生命周期终止**：不发布新的补丁版本

## 建议

- **保持最新**：我们建议升级到最新的次要版本，以获得全面支持并访问新功能
- **规划升级**：关注更新日志以了解即将到来的版本变更，并相应规划升级
- **安全性**：关键安全修复仅提供给活跃支持和关键支持版本
- **测试**：在生产环境升级前，使用新版本测试您的应用程序

## 版本兼容性

在次要版本之间升级时：

- 查看更新日志以了解破坏性变更
- 彻底测试您的应用程序
- 遵循文档中提供的升级指南
- 考虑您当前版本的支持时间线
